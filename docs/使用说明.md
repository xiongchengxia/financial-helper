# 财税票据助手 — 使用说明

本系统用于：**上传发票图片或 PDF** → **自动识别结构化字段** → **合规审核与风险标记** → **生成会计凭证** → **导出 Excel**。

> **免责声明**：系统输出仅供参考，入账与纳税申报责任由企业/会计人员承担。

---

## 1. 环境要求

| 组件 | 要求 |
|------|------|
| Python | 3.10+ |
| Node.js | 18+ |
| 网络 | 可访问 PaddleOCR AI Studio、DeepSeek API |

## 2. 安装与启动

### 2.1 克隆仓库

```bash
git clone https://github.com/xiongchengxia/financial-helper.git
cd financial-helper
```

### 2.2 配置 API 密钥（必做，且勿提交到 Git）

```bash
cd apps/api
cp .env.example .env
```

编辑 `apps/api/.env`，填入：

| 变量 | 说明 | 获取方式 |
|------|------|----------|
| `PADDLEOCR_TOKEN` | PaddleOCR 识别 | [AI Studio PaddleOCR](https://aistudio.baidu.com/) |
| `DEEPSEEK_API_KEY` | 结构化抽取 | [DeepSeek 开放平台](https://platform.deepseek.com/) |

可选配置：

| 变量 | 说明 |
|------|------|
| `COMPANY_BUYER_NAME` | 本公司名称（购买方一致性校验） |
| `COMPANY_BUYER_TAX_ID` | 本公司税号 |
| `REIMBURSE_DEADLINE` | 报销截止日期（如 `2025-12-31`） |
| `SENSITIVE_ITEM_KEYWORDS` | 敏感品名关键词，逗号分隔 |
| `PDF_MAX_PAGES` | PDF 最大页数，默认 10 |
| `PDF_MAX_BYTES` | PDF 大小上限，默认 20MB |

前端（一般无需改）：

```bash
cd apps/web
cp .env.local.example .env.local
# NEXT_PUBLIC_API_BASE=http://localhost:8001
```

### 2.3 启动后端（端口 8001）

```bash
cd apps/api
python3 -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

健康检查：浏览器打开 http://localhost:8001/health ，应返回 `{"status":"ok"}`。

### 2.4 启动前端（端口 3001）

```bash
cd apps/web
npm install
npm run dev
```

浏览器打开：**http://localhost:3001**

## 3. 功能操作指南

### 3.1 上传发票

1. 首页点击 **「选择文件」** 或 **「拍照上传」**
2. 支持格式：
   - 图片：JPG / PNG / WebP（≤10MB）
   - PDF 发票（≤20MB，最多 10 页，自动转图后识别）
3. 上传后自动跳转任务页，识别完成后进入 **发票详情**

### 3.2 查看任务

- 菜单 **「任务」**：查看所有识别任务及状态（pending / processing / completed / failed）
- 失败时查看错误信息；常见原因：`PADDLEOCR_TOKEN` 或 `DEEPSEEK_API_KEY` 未配置

### 3.3 发票详情

- **原始 PDF / 原图**：左侧预览
- **结构化字段**：号码、日期、购销方、金额等
- **合规审核**：
  - **形式规则**（`RULE_INV_*`）：必填、税号、金额勾稽、日期等
  - **整体结果**：规则全过 **且** 无阻断级风险才为「通过」
- **风险标记**：如 `RISK_DUPLICATE`（重复发票）、`RISK_BUYER_MISMATCH` 等

#### 数电电子票说明

**电子发票（普通发票）** 票面通常 **没有** 12 位「发票代码」，仅有 20 位「发票号码」。系统对 `vat_electronic` **不要求** `invoice_code`，属正常情况。

#### 规则全过但整体「未通过」？

请查看 **「风险标记」** 区域。重复上传同一发票号码会触发 **`RISK_DUPLICATE`（阻断）**，用于防重复报销。测试时可 **删除旧文档** 后再上传。

### 3.4 删除发票文档

- **发票列表** 或 **发票详情** 点击 **「删除」**
- 将删除：文档记录、关联会计凭证、原文件（若无其它文档引用）

### 3.5 会计凭证

- 发票识别成功且为发票类型时，**自动生成** 采购进项凭证（借：材料采购 / 进项税，贷：应付账款）
- 菜单 **「凭证」** 查看列表与分录明细

### 3.6 导出

- 菜单 **「导出」** → **下载 Excel** 或 **查看 JSON**
- Excel 含：凭证主表、分录明细、风险明细

## 4. 常见问题

| 现象 | 处理 |
|------|------|
| `PADDLEOCR_TOKEN is not configured` | 配置 `apps/api/.env` 并重启后端 |
| `DEEPSEEK_API_KEY is not configured` | 同上 |
| PDF 任务很快失败 | 检查 PDF 页数、大小限制 |
| 合规未通过但规则全绿 | 查看风险标记，多为重复发票 |
| 前端无法连接 API | 确认后端 8001 已启动，`NEXT_PUBLIC_API_BASE` 正确 |

### 刷新合规（不重新上传）

```bash
curl -X POST http://localhost:8001/v1/documents/{文档ID}/compliance/refresh
```
